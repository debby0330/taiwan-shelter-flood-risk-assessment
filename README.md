# 台灣避難所洪災風險評估分析系統
## Flood Risk Assessment for Shelters in Taiwan

### 🎯 專案概述
本專案利用地理資訊系統(GIS)技術，結合水利署河川圖資與消防署避難收容所資料，建立一套完整的洪災風險評估系統，用以評估台灣各行政區避難所的洪災風險與收容量缺口。

### 📋 分析流程 (7個核心步驟)
1. **資料清理** - 移除無效座標的避難所資料
2. **建立緩衝區** - 創建 500m/1km/2km 三級河川緩衝區
3. **風險評估** - 空間連接分析並指定風險等級
4. **行政區統計** - 按鄉鎮區統計避難所數量與容量
5. **缺口分析** - 安全容量缺口分析與風險排名
6. **視覺化** - 創建互動地圖與長條圖
7. **資料匯出** - 匯出 shelter_risk_audit.json

### ⚙️ 重要參數
- **緩衝區距離**: 高風險 500m, 中風險 1km, 低風險 2km
- **座標系統**: TWD97 (EPSG:3826) 進行分析
- **風險優先**: 高風險 > 中風險 > 低風險 > 無風險
- **容量欄位**: 預計收容人數

### 🔧 技術決策記錄

**座標系統選擇**
- 決定使用 TWD97 (EPSG:3826) 進行距離計算
- 確保緩衝區距離的準確性
- 輸出時轉回 WGS84 (EPSG:4326) 便於使用

**風險評估邏輯**
- 採用「最高風險優先」原則
- 高風險權重 3，中風險權重 2，低風險權重 1
- 避免重複計算，每個避難所只分配一個風險等級

**資料清理策略**
- 移除經緯度為 0、null、超出台灣範圍的記錄
- 台灣座標範圍：經度 119°-123°，緯度 21°-26°
- 資料保留率根據實際資料品質而定

**缺口分析標準**
- 安全避難所：無風險 (NONE) 等級
- 風險區人口：高+中+低風險區收容量
- 缺口標準：安全容量比例 < 100%

### 🐛 問題解決記錄

**TypeError: 資料類型衝突**
- **問題**: 經緯度欄位包含字串資料，無法進行數值比較
- **原因**: 原始 CSV 資料中經緯度可能為混合類型
- **解決**: 使用 `pd.to_numeric(errors='coerce')` 轉換資料類型
- **結果**: 成功清理無效座標並轉換為數值類型

**KeyError: capacity 欄位不存在**
- **問題**: 程式碼使用 'capacity' 欄位，但實際資料中不存在
- **原因**: 實際欄位名稱為「預計收容人數」
- **解決**: 實現智能欄位識別，檢查多種可能的欄位名稱
- **結果**: 自動適應不同的欄位命名方式

**一對多空間連接問題**
- **問題**: 避難所可能同時落在多個緩衝區內
- **原因**: 緩衝區重疊導致重複分類
- **解決**: 使用風險權重比較，只保留最高風險等級
- **結果**: 每個避難所只被分配一個風險等級

**Notebook 檔案過大**
- **問題**: Notebook 檔案大小超過編輯限制
- **原因**: 累積了大量程式碼和輸出
- **解決**: 創建外部程式碼檔案，手動更新 notebook
- **結果**: 成功繼續開發而不受檔案大小限制
